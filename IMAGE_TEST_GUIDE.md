# 图片显示测试指南

## 快速测试步骤

### 1. 启动应用
```bash
npm run tauri dev
```

### 2. 测试正常图片显示

**方法一：截图测试**
1. 按 `Cmd+Shift+5` 打开 macOS 截图工具
2. 选择区域截图
3. 截图会自动保存到剪贴板
4. 打开贴立方应用，应该看到图片在列表中显示

**方法二：复制图片**
1. 在浏览器中右键点击任意图片
2. 选择"复制图片"
3. 打开贴立方应用，查看图片显示

### 3. 检查加载状态

**预期效果**：
- ✅ 小图片直接显示
- ✅ 大图片显示旋转加载动画（可能很快）
- ✅ 加载完成后动画消失，图片显示

### 4. 检查错误处理

**模拟错误场景**：
- 如果图片数据损坏或格式不支持
- 会显示红色错误图标 + "图片加载失败"文字

### 5. 测试图片操作

**点击图片**：
- 图片会被复制到剪贴板
- 可以粘贴到其他应用

**点击保存按钮**：
- 图片保存到下载目录
- 文件名格式：`screenshot_时间戳.jpg`

## 性能测试

### 大图片测试
1. 复制一张大图片（>5MB）
2. 观察加载时间
3. 检查内存占用

**预期**：
- 加载时间 < 2秒
- 没有明显卡顿
- 内存占用合理

### 多图片测试
1. 连续复制多张图片（10张+）
2. 观察列表滚动流畅度
3. 检查内存占用

**预期**：
- 滚动流畅
- 图片加载不阻塞
- 内存不持续增长

## 常见问题排查

### Q1: 图片不显示
**检查**：
1. 查看控制台是否有错误日志
2. 检查图片格式是否支持
3. 尝试重新复制图片

### Q2: 图片显示模糊
**原因**：
- 缩略图尺寸限制（最大 200px 高度）
- 这是正常设计，点击可查看原图

### Q3: 图片加载很慢
**原因**：
- 大图片需要解码时间
- base64 数据量大

**解决**：
- 后端生成缩略图
- 使用图片压缩

### Q4: 错误占位符一直显示
**检查**：
1. 查看控制台日志
2. 确认图片数据是否完整
3. 尝试不同来源的图片
