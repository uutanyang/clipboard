# 草果剪贴板 - 测试快速指南

## 快速开始测试

### 方法一：使用自动化测试脚本（推荐）

```bash
# 运行完整的功能测试套件
chmod +x test-functional.sh
./test-functional.sh
```

该脚本会自动：
- 启动两个应用实例
- 测试 HTTP 服务器
- 检查 mDNS 发现
- 指导完成配对流程
- 验证文本同步
- 验证文件传输
- 生成测试报告

### 方法二：手动测试

## 测试步骤详解

### 1. 启动两个应用实例

#### 方式 A：使用不同端口（同一台机器）

**终端 1 - 启动实例 1：**
```bash
cd /Users/yangtanfang/project/2026/4cool/clipboard-caoguo
npm run tauri dev
```

**终端 2 - 启动实例 2：**
```bash
cd /Users/yangtanfang/project/2026/4cool/clipboard-caoguo

# 修改端口配置
sed -i.bak 's/const DEFAULT_HTTP_SERVER_PORT: u16 = 54321/const DEFAULT_HTTP_SERVER_PORT: u16 = 54322/' src-tauri/src/server/mod.rs

# 启动实例 2
npm run tauri dev
```

**预期结果：**
- ✓ 实例 1 的 HTTP 服务器在端口 54321 启动
- ✓ 实例 2 的 HTTP 服务器在端口 54322 启动
- ✓ 两个应用窗口都显示
- ✓ 系统托盘图标显示

#### 方式 B：使用不同设备（最佳）

- **设备 1**：运行应用，端口 54321
- **设备 2**：运行应用，端口 54321（同一网络，无需修改端口）

**预期结果：**
- ✓ 两个设备在同一局域网
- ✓ 两个应用都正常运行

### 2. 测试 mDNS 设备发现

**步骤：**
1. 在实例 1 中，点击顶部的 **"设备"** 按钮
2. 观察 **"启用设备发现"** 开关是否自动开启
3. 查看设备列表

**预期结果：**
- ✓ "启用设备发现" 开关默认为开启状态
- ✓ 设备列表显示本机设备（状态：已配对）
- ✓ 控制台日志显示 mDNS 服务已启动

**验证命令：**
```bash
# 查看 HTTP 服务器响应
curl http://localhost:54321/ping

# 期望输出：
# {
#   "status": "ok",
#   "hostname": "...",
#   "ip": "...",
#   "version": "0.1.0",
#   "timestamp": "..."
# }
```

### 3. 测试设备互相发现

**步骤：**
1. 确保两个实例都已启动
2. 在实例 1 和实例 2 中都点击 **"设备"** 按钮
3. 等待 5-10 秒

**预期结果：**
- ✓ 实例 1 的设备列表中显示实例 2（状态：未配对）
- ✓ 实例 2 的设备列表中显示实例 1（状态：未配对）
- ✓ 设备信息（名称、IP、端口）正确显示

**如果未发现：**
```bash
# 检查防火墙设置（macOS）
# 系统偏好设置 -> 安全性与隐私 -> 防火墙 -> 防火墙选项
# 确保 mDNS 流量允许

# 检查网络连接
ping <对方设备IP>

# 查看应用日志
tail -f logs/instance1.log | grep -i mdns
tail -f logs/instance2.log | grep -i mdns
```

### 4. 测试配对流程

**步骤：**
1. 在实例 1 的设备列表中，找到实例 2
2. 点击 **"配对"** 按钮
3. 在实例 2 中，查看是否收到配对请求通知/对话框
4. 在实例 2 中点击 **"接受"** 按钮
5. 查看两个实例的设备列表状态

**预期结果：**
- ✓ 实例 2 收到配对请求提示
- ✓ 点击接受后，两个设备状态都变为 **"已配对"**
- ✓ 实例 1 的设备列表：实例 2（状态：已配对）
- ✓ 实例 2 的设备列表：实例 1（状态：已配对）

**验证：**
```bash
# 查看日志中的配对信息
tail -f logs/instance1.log | grep -i pair
tail -f logs/instance2.log | grep -i pair
```

---

### 5. 测试文本同步

#### 5.1 单向同步

**步骤：**
1. 在实例 1 的系统剪贴板中复制文本：
   ```bash
   echo "测试文本同步 - $(date)" | pbcopy
   ```

2. 在实例 1 的应用中查看剪贴板历史

3. 等待 3-5 秒

4. 在实例 2 的应用中查看剪贴板历史

**预期结果：**
- ✓ 实例 1 的剪贴板历史显示刚复制的文本
- ✓ 实例 2 的剪贴板历史也显示同样的文本
- ✓ 时间戳一致

#### 5.2 双向同步

**步骤：**
1. 在实例 2 复制文本：
   ```bash
   echo "从实例B同步 - $(date)" | pbcopy
   ```

2. 等待 3-5 秒

3. 在实例 1 中查看剪贴板历史

4. 在实例 1 复制文本：
   ```bash
   echo "从实例A同步 - $(date)" | pbcopy
   ```

5. 等待 3-5 秒

6. 在实例 2 中查看剪贴板历史

**预期结果：**
- ✓ 实例 1 收到实例 2 的剪贴板内容
- ✓ 实例 2 收到实例 1 的剪贴板内容
- ✓ 双向同步正常工作
- ✓ 没有重复内容

**验证：**
```bash
# 查看同步日志
tail -f logs/instance1.log | grep -i "clipboard\|sync"
tail -f logs/instance2.log | grep -i "clipboard\|sync"
```

### 6. 测试文件传输

#### 6.1 小文件传输（1MB）

**步骤：**
1. 创建测试文件：
   ```bash
   echo "测试文件内容 - $(date)" > /tmp/test-small.txt
   ```

2. 在实例 1 中点击 **"文件"** 按钮
3. 点击 **"选择文件"** 按钮
4. 选择 `/tmp/test-small.txt`
5. 选择目标设备：实例 2
6. 点击 **"发送"** 按钮
7. 观察传输进度
8. 传输完成后，在实例 2 中查看文件传输历史
9. 在实例 2 的下载目录中查找文件：
   ```bash
   open ~/Downloads
   ```

**预期结果：**
- ✓ 文件选择正常
- ✓ 传输进度显示正常
- ✓ 传输成功完成
- ✓ 实例 2 的传输历史显示接收记录
- ✓ 文件正确保存到下载目录
- ✓ 文件内容完整

#### 6.2 中等文件传输（10MB）

**步骤：**
1. 创建 10MB 测试文件：
   ```bash
   dd if=/dev/urandom of=/tmp/test-medium.bin bs=1M count=10
   ```

2. 在实例 1 中发送该文件到实例 2
3. 观察传输进度

**预期结果：**
- ✓ 传输成功
- ✓ 进度条实时更新
- ✓ 内存占用正常

#### 6.3 大文件传输（50MB，可选）

**步骤：**
1. 创建 50MB 测试文件：
   ```bash
   dd if=/dev/urandom of=/tmp/test-large.bin bs=1M count=50
   ```

2. 发送并观察传输过程

**预期结果：**
- ✓ 大文件传输成功
- ✓ 没有内存溢出
- ✓ 传输速度合理

**验证：**
```bash
# 查看 MD5 校验
md5 /tmp/test-small.txt
md5 ~/Downloads/test-small.txt

# 两个 MD5 值应该相同
```

### 7. 测试离线重连

**步骤：**

**预期结果：**
- ✓ 实例 2 关闭后，实例 1 检测到设备离线
- ✓ 实例 2 重启后，实例 1 自动发现设备上线

## 问题排查

### 问题：无法发现其他设备

**可能原因：**
1. 防火墙阻止 mDNS 流量
2. 设备不在同一局域网
3. mDNS 服务未启动

**解决方案：**
```bash
# 检查网络连接
ping <对方设备IP>

# 测试 HTTP 连接
curl http://<对方设备IP>:54321/ping

# 检查应用日志
tail -f logs/instance1.log | grep -i "mdns\|discover"
```

### 问题：配对请求未收到

**解决方案：**
```bash
# 测试 HTTP 连接
curl -X POST http://<对方设备IP>:54321/api/pair \
  -H "Content-Type: application/json" \
  -d '{"device_id":"...","device_name":"..."}'

# 检查日志
tail -f logs/instance1.log | grep -i "pair"
```

### 问题：剪贴板不同步

**解决方案：**
- 检查设备是否已配对
- ping 对方设备确认在线
- 查看同步服务日志
