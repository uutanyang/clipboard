# 使用说明

## 安装和运行

### 1. 安装依赖

```bash
npm install
```

### 2. 运行开发版本

```bash
npm run tauri dev
```

这将启动：
- Vite 开发服务器（端口 1421）
- Tauri 应用窗口

### 3. 构建生产版本

```bash
npm run tauri build
```

构建产物位于 `src-tauri/target/release/bundle/` 目录

## 功能说明

### 自动监听剪贴板

应用会在后台自动监听剪贴板变化。每当你复制新的文本内容时，它会自动保存到数据库中。

### 历史记录查看

- 所有剪贴板记录会按时间倒序排列
- 每条记录显示：
  - 内容类型图标（📝 文本 / 🖼️ 图片）
  - 复制时间
  - 内容预览

### 搜索功能

- 在搜索框输入关键词即可搜索历史记录
- 支持模糊搜索
- 搜索结果实时显示

### 快捷键操作

| 快捷键 | 功能 |
|--------|------|
| `Cmd/Ctrl + Shift + V` | 呼出/隐藏剪贴板面板 |
| `↑` / `↓` | 上下选择记录 |
| `Enter` | 复制选中的内容到剪贴板 |
| `Esc` | 隐藏面板 |

### 管理记录

- **删除单条记录**: 点击记录右侧的 ✕ 按钮
- **清空所有记录**: 点击搜索栏右侧的"清空"按钮

## 技术架构

### 后端（Rust）

- **Tauri 2.0**: 桌面应用框架
- **rusqlite**: SQLite 数据库
- **arboard**: 跨平台剪贴板访问
- **chrono**: 时间处理

主要功能：
- 剪贴板监听（每 500ms 检查一次）
- 数据库 CRUD 操作
- 全局快捷键管理
- 前后端通信

### 前端（Vue 3）

- **Vue 3**: 前端框架
- **TypeScript**: 类型安全
- **Vite**: 构建工具

主要功能：
- 剪贴板记录列表展示
- 实时搜索
- 键盘导航
- 响应式 UI

## 数据存储

剪贴板数据存储在本地的 SQLite 数据库中：

**数据库位置**: `clipboard.db`

**表结构**:
```sql
CREATE TABLE clipboard_items (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    content_type TEXT NOT NULL,
    content TEXT NOT NULL,
    created_at TEXT NOT NULL
)
```

## 注意事项

1. **首次运行**: 首次运行时会自动创建数据库文件
2. **隐私**: 所有数据存储在本地，不会上传到任何服务器
3. **性能限制**: 为保证性能，最多保留 100 条历史记录
4. **文本格式**: 目前仅支持纯文本格式，富文本和图片功能待开发

## 常见问题

### Q: 快捷键不工作？

A: 确保 Tauri 应用正在运行。快捷键仅在应用运行时生效。

### Q: 剪贴板记录丢失？

A: 检查 `clipboard.db` 文件是否存在于项目目录中。

### Q: 如何更改快捷键？

A: 修改 `src-tauri/src/lib.rs` 中的快捷键配置，然后重新编译。

### Q: 如何调整历史记录数量限制？

A: 修改 SQL 查询中的 `LIMIT 100` 为你想要的数值。

## 开发建议

### 添加新功能

1. **图片支持**: 扩展 `content_type` 为 "image"，使用 base64 编码
2. **富文本**: 识别并存储富文本格式
3. **云同步**: 添加后端 API 进行数据同步
4. **主题系统**: 添加亮色/暗色主题切换

### 调试

- **前端调试**: 使用浏览器开发者工具（F12）
- **后端调试**: 使用 `println!` 或 Rust 调试器
